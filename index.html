Week 4: Learning CSS Basics

Created on October 28, 2020

In this week, we start to add some life and color to the website. With folder that we created on first week called CSS, we are finally going to give some meaning to those initials. W3.org describes CSS as:

Cascading Style Sheets (CSS) is a simple mechanism for adding style (e.g., fonts, colors, spacing) to Web documents.

www.w3.org

The reason that we kept the HTML and the CSS in different files is to avoid confusion as we add content more each week. However, we needed to find a way to link them together by using the tag below:


<link rel="stylesheet" href="css/style.css">
rel
To inform web document that it's file is linked to style sheet
href
To locate stylesheet

Now that we can controlled the style of the page, I choose new font for body and different one for the headers from Abode Font.

In order to connect the style with HTML element, we have to use the proper selector. E.g:
code { font-size: 18px; font-weight: 700; white-space: pre; }

From CSS-Tricks, I learned that I can target specific elements and create rules for different groups using the selectors below:

  • #id: use for unique element on website.
  • .class: use for all the elements of the same type.

I came with the conclusion that without CSS, website is just simple and dull text.

Week 6: Making page responsive

Created on November 11, 2020

As CSS file start to grow, i realize how important is to keep your stylesheet organized and tidy.I found few tips to keep well organized:

Keep it consistent
The first and more important to keep everything in same formatting such as naming the classes, decide between use rem or em or how to name the color with HEX or RBG.
Making comment on CSS
Adding comments to make easy to find code after and make easier for others to understand what code is doing.
Make logical order for stylesheet
Decide to start with general styles, after classes and at end media queries.

On this week, we learned how to make page responsive with CSS tricks. I made the decision to go for Holy Grail Layout:

Holy Grail is a layout pattern that's very common on the web. It consists of a header, a main content area with fixed-width navigation on the left, content in the middle and a fixed-width sidebar on the right and then a footer.

CSS Grid: Holy Graid Layout

At first, I had position the sidebar on mobile version to be float to the right. As left-handed user myself, the struggle to avoid clicking the like button while scroll down to the page on mobile is inevitable.

However, after i found article that said 49% of users are one-handed use in which 67% use their right thumb on the screen to navigate the website. I had to relocate my sidebar to the top to increase usability. Also relocated "go to top" button to left when using mobile devices.

On the same article, shows that holding mobile device is landscape mode is about 10% for that reason, i took it out the sidebar menu from landscape mode.

Week 9: Forms

Created on December 1, 2020

On this week, we learned how to create contact form with HTML and CSS. I decide to focus more HTML part and keep CSS clean and simple.

Kept form simple only asked relevant information. Learned to avoid dropdown menu when it is possible because it does not work well in mobile version and to opt for radio input instead and to keep the label and the input close to each other making easier recognition.

On the web, people’s attention spans are very short. Your visitors will tune out your content if you don’t get to the point quickly. According to a Microsoft Research study by Chao Liu and colleagues, users will leave a page in just 10 to 20 seconds, not leaving designers with much time to successfully persuade their visitors to take a specific action.

The Dos and Don’ts of Designing Web Forms

There are essential tags to create form:

<form id="form">
First step and involves all the form.id attribute is essential for interactive of the webpage.
<fieldset>
Divide the form with different groups of boxes.
<legend>
Where you name the different groups.
<label for="name">
Label for input data.for attribute needs to be equal as id attribute of the input.
<input id="name">
There are few type of inputs however the most important ones being: text for enter data and submit to the form.
<textarea>
Usually used for message.
<select>
Create select options box.
